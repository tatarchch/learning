# Go Learning Project

## О проекте

Этот репозиторий представляет собой практический путь изучения языка Go и современных подходов к разработке backend-приложений.

Цель проекта — освоить идиоматичный Go-код, понять особенности работы языка и научиться создавать простые, надёжные и поддерживаемые сервисы.

В репозитории будут собраны практические задания, эксперименты с различными возможностями языка и небольшие приложения, созданные в процессе обучения.

---

## Цели обучения

* Изучить философию и основные принципы Go.
* Научиться проектировать понятный и поддерживаемый код.
* Понять особенности работы памяти, значений и указателей.
* Освоить работу со `slice`, `map`, строками и другими базовыми структурами данных.
* Изучить конкурентное программирование с использованием `goroutines`, `channels` и пакета `sync`.
* Научиться правильно использовать `context` и управлять жизненным циклом приложения.
* Создавать HTTP-сервисы с использованием стандартной библиотеки Go.
* Работать с конфигурацией, файловой системой и JSON.
* Изучить работу с базами данных через `database/sql`.
* Освоить написание unit, integration и benchmark тестов.

---

## Темы обучения

### Основы языка и проектирование

* Структуры и методы.
* Интерфейсы и композиция.
* Организация пакетов.
* Обработка ошибок.
* Принципы проектирования приложений на Go.

### Работа с данными и памятью

* Value и pointer semantics.
* Устройство и особенности `slice`.
* Работа с `map`.
* Строки, `byte` и `rune`.
* Производительность и управление памятью.

### Конкурентность

* Goroutines.
* Channels.
* Select.
* Паттерны конкурентного программирования.
* Пакет `sync`.

### Backend-разработка

* HTTP серверы и обработчики.
* Middleware.
* Работа с JSON.
* Конфигурация приложения.
* Организация структуры проекта.
* Dependency Injection через явное создание зависимостей.

### Работа с базой данных

* `database/sql`.
* Repository pattern.
* Миграции и организация доступа к данным.

### Тестирование

* Table-driven tests.
* Test doubles.
* Integration tests.
* Race detector и Benchmark.
